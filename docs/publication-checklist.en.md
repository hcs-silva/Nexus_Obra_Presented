# Publication Checklist

## Content

- [ ] Review wording for external public audience.
- [ ] Keep balance between product messaging and technical depth.
- [ ] Ensure terminology and heading consistency.

## Security and privacy

- [ ] Confirm absence of secrets, tokens, and credentials.
- [ ] Confirm absence of real personal data and sensitive IDs.
- [ ] Confirm screenshots do not expose critical information.

## Visual quality

- [ ] Validate layout on desktop and mobile.
- [ ] Verify contrast and readability.
- [ ] Ensure all images load correctly.

## Publishing

- [ ] Commit the Nexus_Portefólio folder.
- [ ] Configure GitHub Pages for the correct branch/folder.
- [ ] Test internal links after deployment.
- [ ] Perform final navigation and spelling review.
