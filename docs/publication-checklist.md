# Publication Checklist

## Conteúdo

- [ ] Revisar linguagem para público externo.
- [ ] Garantir equilíbrio entre mensagem de produto e profundidade técnica.
- [ ] Verificar consistência de termos e títulos.

## Segurança e Privacidade

- [ ] Confirmar ausência de secrets, tokens e credenciais.
- [ ] Confirmar ausência de dados pessoais e IDs reais sensíveis.
- [ ] Confirmar que screenshots não expõem informação crítica.

## Qualidade Visual

- [ ] Validar layout em desktop e mobile.
- [ ] Verificar contraste e legibilidade.
- [ ] Garantir que imagens carregam corretamente.

## Publicação

- [ ] Commit da pasta Nexus_Portefólio.
- [ ] Configurar GitHub Pages para branch/pasta correta.
- [ ] Testar links internos após deploy.
- [ ] Fazer revisão final de navegação e ortografia.
